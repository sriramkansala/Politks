# Reference Screenshots

Visual training data for this repository's design taste.

- `approved/` — screens whose **principles** (hierarchy, spacing, density,
  typography, interaction, state design) we want to learn from.
- `rejected/` — screens that demonstrate anti-patterns to avoid.

## Rules

1. **Never add an unannotated screenshot.** For every image dropped here,
   create a matching note in `../reference-notes/` using `_TEMPLATE.md`. A
   screenshot without explanation teaches the wrong lesson.
2. **Borrow principles, not branding.** We extract hierarchy/spacing/density
   lessons — we never copy another product's logo, color, or proprietary
   layout.
3. Name files descriptively: `linear-issue-list.png`, `stripe-onboarding-
   step-2.png`, not `Screenshot 2026-05-30.png`.
